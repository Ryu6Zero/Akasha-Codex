## Before Publishing

Run the full local gate:

```bash
npm run prepare:open-source
```

Then run the GitHub release readiness check:

```bash
npm run release:check
```

This command fails if the working tree is dirty, forbidden local data is tracked, `origin` is missing, the version tag is missing or does not point at `HEAD`, or GitHub CLI is not logged in.

Confirm the repository is clean:

```bash
git status --short
```

Confirm private or generated data is not tracked:

```bash
git ls-files | Select-String -Pattern '^(library/|config/|release/|dist/|node_modules/|\.env|\.local-appdata/|\.npm-cache/|android/app/src/main/assets/public/|docs/progress\.md|docs/project-memory\.md)'
```

The command above should print nothing.

## Create The GitHub Repository

Create an empty GitHub repository, then add it as `origin`:

```bash
git remote add origin https://github.com/<owner>/<repo>.git
npm run release:check
git push -u origin main
```

For this project, the expected repository name is `Akasha-Codex`.

## Publish A Version

Use semantic version tags that match `package.json`.

```bash
git tag -a v0.1.0 -m "ç»¯ه…¸éکپ 0.1.0"
npm run release:check
git push origin v0.1.0
```

Pushing a `v*` tag triggers `.github/workflows/windows-portable.yml`.
That workflow builds the Windows portable executable and attaches it to the GitHub Release.

## Manual Local Package

To build locally:

```bash
npm run package:win
```

The portable executable is written to:

```text
release/ç»¯ه…¸éکپ 0.1.0.exe
```

Do not commit `release/`. Attach the executable to GitHub Release only.

## Release Privacy Checklist

- Do not upload `library/`.
- Do not upload `config/`.
- Do not upload screenshots containing personal local paths.
- Do not upload downloaded third-party asset packs as source repository content.
- Do not upload `.env*`, API keys, credentials, or private notes.
- Keep `docs/progress.md` and `docs/project-memory.md` local.

## First Release Recommendation

For `v0.1.0`, publish:

- Source code generated by GitHub automatically.
- `ç»¯ه…¸éکپ 0.1.0.exe` from the Windows portable workflow.

Do not publish:

- The local `library/` directory.
- Full imported third-party assets or generated user libraries.
- The local `config/` directory.
