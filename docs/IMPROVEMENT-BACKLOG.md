# 绯典阁改进路线图

这份文档用于下一轮产品讨论。当前版本已经能用，但还没到“长期维护舒服”的程度。下一轮不应继续堆视觉糖，应该先把资料库治理、资产维护和移动端体验做扎实。

## 讨论方式

每个主题都需要先定方向，再进入开发。不要一边写代码一边猜产品规则。

- 是否进入下一轮：做 / 不做 / 延后
- 做到什么程度：最小可用 / 完整方案
- 风险边界：是否会增加资料库体积、是否会误删用户资产、是否会引入版权问题

## P0：建议下一轮优先

### 1. 标签治理中心

问题：当前分类分组已经改成选择式标签，但标签仍然散落在角色编辑、快捷筛选和分组规则里。长期看会出现大小写重复、同义词重复、删除标签后规则失效等问题。

建议：做一个“标签管理”页，支持重命名、合并、删除、批量替换。分组规则、角色标签、搜索筛选都从同一份标签索引读取。

最小可用：显示全库标签、使用次数、失效规则；支持合并两个标签和删除未使用标签。

验收：删除标签后，分组规则里不会留下不可见条件；合并标签后，角色和分组筛选结果一致刷新。

### 2. 资产体积与完整性报告

问题：头像、立绘、模型、语音和附件长期积累后会迅速膨胀。用户需要知道哪些角色占用空间最大、哪些文件缺失、哪些文件未被引用。

建议：增加后台报告命令或设置页报告，输出每个角色的资产数量、缺失资产、孤儿文件、估算体积和最大体积角色排行。

最小可用：输出 JSON 和 Markdown 两份报告。

验收：报告能列出总角色数、总立绘数、总模型数、缺失数量、孤儿文件数量、最大体积角色 Top 20。

### 3. 详情页资料结构化

问题：详情页文本排版已经修过，但复杂角色资料本质上是结构化数据。继续把来源、职阶、稀有度、CV、画师、设定条目都堆进人物介绍，后面一定难维护。

建议：新增 `profileFields` 或 `officialProfile` 字段，把来源、分类、稀有度、CV、画师、属性、资料条目拆开显示。人物介绍保留为用户可编辑摘要。

最小可用：详情页以字段表和分段资料展示结构化信息。

验收：详情页能清晰区分“结构化字段”和“用户备注”，再次导入或编辑不会覆盖用户手写介绍。

## P1：体验增强

### 4. 分类规则支持 AND / OR

问题：目前分类规则是任一标签命中即可。比如想做“某来源 + 某职业 + 某稀有度”这样的筛选，就不够精确。

建议：分类规则增加模式：任一匹配 OR、全部匹配 AND。UI 上用分段控件切换。

验收：同一组标签在 OR 和 AND 模式下结果数量不同且符合预期。

### 5. 资产批量操作

问题：头像、立绘、附件多起来后，逐个删除和设置封面会很累。

建议：资产区支持多选、批量删除、批量重命名、按来源筛选、批量设置排序。

验收：至少能一次删除多张非主头像/非封面立绘，并保留主头像和封面兜底逻辑。

### 6. 资料库健康检查页

问题：现在用户不知道资料库有没有孤儿文件、缺失资产、坏 JSON、无效路径、占用空间异常。

建议：设置页新增“资料库健康”tab，显示角色数、资产数、空间占用、坏记录、孤儿文件和一键修复建议。

验收：能检测坏 JSON、缺失资产引用、未被引用的孤儿文件，并输出修复前预览。

## P2：长期方向

### 7. Android 移动布局

问题：Android 端已经有 Capacitor 架构，但桌面密集布局不适合手机。

建议：移动端做独立信息架构：底部导航、单列卡片、详情页分段、资产管理抽屉。

验收：手机竖屏下不用横向滚动也能完成创建角色、导入头像、编辑介绍、查看大图。

### 8. 大图预览胶片栏

问题：现在大图能缩放拖拽，但多图切换还不够直观。

建议：加底部缩略图胶片栏，支持点击跳图和键盘左右切换。

验收：多张立绘详情中能快速定位某张图，缩放切图后状态按预期重置。

### 9. 语音资料收藏与播放列表

问题：语音现在是资料资产，缺少“常用收藏/分类播放”的资料库体验。

建议：语音支持收藏、分类、连续播放、备注搜索。

验收：用户能按分类播放语音，并收藏常用语音。

## 开源前建议

- README、LICENSE、CONTRIBUTING、PRIVACY、Issue/PR 模板
- `npm run prepare:open-source`
- GitHub CI 与 Windows portable 构建工作流

- GitHub 仓库名和 owner
- 是否公开当前 Android 工程
- 首个 Release 是否只放 portable exe
- README 是否需要截图，截图必须使用可公开素材
