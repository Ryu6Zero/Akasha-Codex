# Data Format

绯典阁 is a local character encyclopedia. Catalog metadata lives beside character entries so the whole library can be copied as a folder.

## Library

```text
library/
  catalog.json
  story-catalog.json
  catalog-assets/
    wallpapers/
    icons/
  characters/
    <character-id>/
      character.json
      avatar/
      portraits/
      voices/
      attachments/
      models/
  stories/
    <story-id>/
      story.json
      images/
```

## Catalog Metadata

```ts
type CatalogMetadata = {
  wallpaperPath?: string;
  defaultCollectionId: string;
  defaultSortMode: 'updatedAt' | 'createdAt' | 'name' | 'sourceTitle';
  collections: CatalogCollection[];
};

type CatalogCollection = {
  id: string;
  name: string;
  description: string;
  iconPath?: string;
  tagRules: string[];
};
```

## Character

```ts
type Character = {
  id: string;
  name: string;
  sourceTitle: string;
  aliases: string[];
  tags: string[];
  collectionIds: string[];
  description: string;
  notes: string;
  avatarPath?: string;
  avatarPaths?: string[];
  portraitPath?: string;
  portraitPaths?: string[];
  voiceAssets: VoiceAsset[];
  voicePaths: string[];
  modelPath?: string;
  modelFileName?: string;
  modelFormat?: string;
  attachmentPaths: string[];
  createdAt: string;
  updatedAt: string;
};

type VoiceAsset = {
  id: string;
  label: string;
  line?: string;
  filePath: string;
  fileName?: string;
};
```

`avatarPath` is the primary avatar. `avatarPaths` stores all imported avatars. `portraitPath` is the cover portrait. `portraitPaths` stores all imported portraits. Older entries with only `avatarPath` or `portraitPath` remain readable.

Renderer-only URL fields such as `avatarUrl`, `avatarUrls`, `portraitUrls`, and `voiceAssets[].fileUrl` are generated by the active platform client at read time and are not persisted.

## Story Archive

```ts
type StoryCatalogMetadata = {
  defaultCategoryId: string;
  defaultSortMode: 'updatedAt' | 'createdAt' | 'title';
  categories: StoryCategory[];
};

type StoryCategory = {
  id: string;
  name: string;
  description: string;
  tagRules: string[];
};

type Story = {
  id: string;
  title: string;
  subtitle: string;
  summary: string;
  categoryIds: string[];
  tags: string[];
  linkedCharacterIds: string[];
  blocks: StoryBlock[];
  coverImagePath?: string;
  createdAt: string;
  updatedAt: string;
};

type StoryBlock = {
  id: string;
  type: 'heading' | 'paragraph' | 'quote' | 'image';
  text: string;
  imagePath?: string;
  imageFileName?: string;
  caption?: string;
};
```

Story text supports encyclopedia links using `[[Character Name]]`. The renderer resolves links against character id, name, and aliases. Backlinks are derived from story records and are not written into `character.json`.

Renderer-only URL fields such as `coverImageUrl` and `blocks[].imageUrl` are generated by the active platform client at read time and are not persisted.

## Android Library

The Android app uses the same logical library format inside the Capacitor app data directory. It does not read the desktop `config` or `library` folders unless a future import/export flow is added.

## Model Policy

Models remain archive attachments only. The active catalog UI does not include a 3D preview surface or Three.js dependency.
